# 每日清单与备忘录 - 宝塔面板部署方案

## 一、架构说明

```
用户浏览器
    ↓
Nginx (宝塔) → 反向代理
    ↓
Node.js 后端 (端口 3000)
    ├── /api/*   → API 接口
    └── /*       → 静态文件 (index.html, css, js)
    ↓
MySQL 数据库
```

---

## 二、服务器要求

- 云服务器：阿里云 / 腾讯云 / 华为云 等（1 核 1G 起即可）
- 系统：CentOS 7+ 或 Ubuntu 18+
- 已安装宝塔面板（bt.cn）

## 三、宝塔面板准备

### 1. 安装必要软件

在宝塔「软件商店」中安装：

| 软件 | 版本 | 说明 |
|------|------|------|
| Nginx | 1.20+ | Web 服务 |
| MySQL | 5.7+ 或 8.0 | 数据库 |
| Node 版本管理器 | 最新 | 管理 Node.js |
| PM2 管理器 | 最新 | 托管 Node 进程 |

### 2. 创建 MySQL 数据库

> **若提示「数据库用户名或密码错误」**，请先完成下方「重置 MySQL root 密码」后再操作。

1. 进入 **数据库** → **添加数据库**
2. 数据库名：`daily_checklist`
3. 用户名、密码：自行设置并记住
4. 进入 **phpMyAdmin**（点击该数据库右侧的「管理」）：
   - **方式 A - 导入文件（推荐）**：点击顶部「导入」→ 选择本地的 `server/db.sql` 文件 → 执行
   - **方式 B - 粘贴 SQL**：点击「SQL」标签 → 打开本地 `server/db.sql` 文件，复制其中全部 SQL 语句 → 粘贴到输入框 → 执行
5. ⚠️ 不要把 `server/db.sql` 这几个字当命令输入，必须导入该文件或粘贴其内容

#### 重置 MySQL root 密码（报错时操作）

**方法一：宝塔面板内重置（推荐）**

1. 进入宝塔 **数据库** 页面
2. 找到 **root 密码** 或 **重置 root 密码** 入口（通常在数据库列表上方）
3. 设置新密码并提交
4. 若入口不可用，进入 **软件商店** → 找到 **MySQL** → **设置** → **root 密码** 修改

**方法二：SSH 命令行**

```bash
# 1. 停止 MySQL
/etc/init.d/mysqld stop

# 2. 跳过授权表启动（宝塔安装的 MySQL 路径可能为 /www/server/mysql）
mysqld_safe --skip-grant-tables &
sleep 3

# 3. 无密码登录并修改（MySQL 5.7）
mysql -u root <<EOF
USE mysql;
UPDATE user SET authentication_string=PASSWORD('你的新密码') WHERE User='root';
FLUSH PRIVILEGES;
EXIT;
EOF

# 若为 MySQL 8.0，改用：ALTER USER 'root'@'localhost' IDENTIFIED BY '你的新密码';

# 4. 重启 MySQL
/etc/init.d/mysqld restart
# 或
killall mysqld
/etc/init.d/mysqld start
```

修改完成后，回到宝塔数据库页面重新创建数据库。

## 五、配置 Node 后端

### 1. 安装依赖

> **若报错 `npm: command not found`**：先进入宝塔 **软件商店** → **Node 版本管理器** → **设置** → 在顶部「命令行版本」中选中已安装的 Node 版本并保存，使终端能识别 `npm`。

在宝塔「终端」或 SSH 中执行：

```bash
cd /www/wwwroot/daily-checklist/server
npm install
```

### 2. 配置数据库连接

编辑 `server/index.js` 中的 `dbConfig`，或使用环境变量。推荐在宝塔 PM2 中配置环境变量：

| 变量名 | 说明 | 在宝塔中的查看位置 |
|--------|------|-------------------|
| DB_HOST | 数据库主机 | 本地数据库固定为 `localhost` |
| DB_PORT | 端口 | 软件商店 → MySQL → 设置 → 端口（默认 3306） |
| DB_USER | 用户名 | 数据库 → 你添加的 `daily_checklist` 库 → 对应「用户名」 |
| DB_PASSWORD | 密码 | 添加数据库时设置的密码；若忘记可点该库「改密」重置 |
| DB_NAME | 数据库名 | 你创建的库名，本例为 `daily_checklist` |
| PORT | 服务端口 | 固定 3000，或自定 |

### 3. 使用 PM2 启动

> **路径说明**：下面示例中的 `/www/wwwroot/daily-checklist/` 需改为你实际上传的目录。若不确定，在终端执行 `ls /www/wwwroot/` 查看实际目录名。

**方式一：PM2 管理器网页界面**

1. 宝塔 → **软件商店** → **PM2 管理器** → 打开
2. 点击 **添加项目**：
   - 启动文件：`/www/wwwroot/你的目录名/server/index.js`（替换为实际路径）
   - 项目名称：`daily-checklist-api`
   - 运行目录：`/www/wwwroot/你的目录名/server`
3. 启动并设置「开机自启」

**若报错 `pm2: command not found`，按以下步骤排查：**

1. 在 PM2 管理器中**切换 Node 版本**到 v16 或以上（如 v18、v20）
2. SSH 或宝塔终端执行：
   ```bash
   ls /www/server/nvm/versions/node/
   ```
   查看实际安装的版本号（如 `v18.20.0`）
3. 编辑 root 的 `~/.bash_profile`，在末尾添加（**把 v18.20.0 换成你看到的版本号**）：
   ```bash
   PATH=$PATH:/www/server/nvm/versions/node/v18.20.0/bin
   ```
4. 执行 `source ~/.bash_profile` 使生效
5. 重新在 PM2 管理器中添加项目

**方式二：命令行**

本项目已把 pm2 纳入依赖，安装后直接用即可（用 `node` 调用，避免 npx 路径问题）：

```bash
cd /www/wwwroot/你的目录名/server   # 先确认该路径存在
npm install                        # 会一并安装 pm2
npm run pm2:start                  # 或：node ./node_modules/pm2/bin/pm2 start index.js --name daily-checklist-api
node ./node_modules/pm2/bin/pm2 save
node ./node_modules/pm2/bin/pm2 startup
```

> **若 `npm install` 报 404**：切回官方源后重试：
> ```bash
> npm config set registry https://registry.npmjs.org/
> npm install
> ```

**方式三：改用「网站 - Node 项目」（宝塔 8.0+ 推荐）**

若 PM2 管理器始终报错，可使用宝塔自带的 Node 项目部署：
1. 网站 → 添加站点（或选择已有站点）→ 设置
2. 左侧找到 **Node 项目**，添加项目，指定启动文件 `server/index.js`、端口 3000
3. 运行目录选择 `server` 所在目录

## 六、配置 Nginx 反向代理

### 1. 添加站点

1. 宝塔 → **网站** → **添加站点**
2. 域名：填写你的域名，或先用 IP
3. 根目录：可随意（后面会改）
4. PHP：选择「纯静态」
5. 创建

### 2. 修改站点配置

进入该站点 → **设置** → **反向代理** → **添加反向代理**：

- 代理名称：`daily-checklist`
- 目标 URL：`http://127.0.0.1:3000`
- 发送域名：`$host`

保存后，再进入 **配置文件**，确保有类似配置：

```nginx
location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_cache_bypass $http_upgrade;
}
```

### 3. 绑定域名 / SSL（可选）

- 域名：在站点设置中绑定
- SSL：申请 Let's Encrypt 免费证书并开启 HTTPS

## 八、代码更新（推荐用部署脚本）

> 国内服务器访问 GitHub 常不稳定，**不推荐**用 Git 拉取。推荐用下方部署脚本，本地一条命令直接同步到服务器。

### 使用部署脚本

**首次配置：**

1. 项目里已有 `deploy.config.example`，复制为 `deploy.config` 并填写：
   ```
   REMOTE_USER=root
   REMOTE_HOST=121.41.198.212
   REMOTE_PATH=/www/wwwroot/daily-checklist/daily-checklist-app
   ```
2. 配置 SSH 免密（可选，否则每次需输密码）：
   ```powershell
   ssh-copy-id root@121.41.198.212
   ```
   若 Windows 没有 `ssh-copy-id`，可用：
   ```powershell
   type $env:USERPROFILE\.ssh\id_rsa.pub | ssh root@121.41.198.212 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
   ```

**日常更新：** 改完代码后，在项目目录执行

- **Windows PowerShell**：`.\deploy.ps1`
- **Mac / Linux / Git Bash**：`./deploy.sh`

脚本会：上传代码 → `npm install` → `pm2 restart`。

## 九、常见问题

| 问题 | 处理 |
|------|------|
| 502 Bad Gateway | 检查 Node 服务是否运行：`pm2 list` |
| 数据库连接失败 | 核对 `dbConfig` 或环境变量，确认 MySQL 已启动 |
| 同步失败 | 检查浏览器控制台，确认 API 地址正确（同域名无需改 config.js） |
| 静态资源 404 | 确认 server 中 `express.static` 指向父目录 |
