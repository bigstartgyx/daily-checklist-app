# 每日清单与备忘录 - 部署方案

> **云服务器 + 宝塔部署**（含后端和数据库）请查看 → [宝塔部署方案.md](./宝塔部署方案.md)

## 一、当前数据存储位置

| 存储项 | 位置 | 说明 |
|--------|------|------|
| 清单、备忘录 | **浏览器 localStorage** | 数据仅存在于用户本地，换设备/清缓存会丢失 |
| 选中日期、Tab | 浏览器 localStorage | 同上 |

**结论**：当前是纯前端应用，无后端，数据不会上传到任何服务器。

---

## 二、部署方式对比

| 方式 | 数据存储 | 适用场景 | 成本 |
|------|----------|----------|------|
| **方式 A** 纯静态托管 | localStorage | 个人/演示用，不要求多端同步 | 低/免 |
| **方式 B** 静态 + 云数据库/API | 云端数据库 | 多设备同步、数据备份 | 中 |

---

## 三、方式 A：纯静态部署（当前即可用）

无需改代码，直接上传 `daily-checklist-app` 文件夹即可。数据仍保存在各用户浏览器。

### 国内云方案

| 服务商 | 产品 | 说明 | 约价 |
|--------|------|------|------|
| **阿里云** | 对象存储 OSS + CDN | 静态文件托管，可绑自定义域名 | OSS 约 0.12 元/GB/月 |
| **腾讯云** | 对象存储 COS + CDN | 同上 | 类似 |
| **华为云** | 对象存储 OBS + CDN | 同上 | 类似 |
| **七牛云** | 对象存储 + CDN | 有免费额度 | 10GB 免费 |
| **Vercel / Netlify** | 静态托管 | 国外服务，可绑域名 | 有免费 tier |

### 推荐步骤（以阿里云 OSS 为例）

1. 开通 **阿里云 OSS**
2. 创建 Bucket（建议选「公共读」）
3. 将 `index.html`、`css/`、`js/` 上传到 Bucket
4. 开启静态网站托管，首页设为 `index.html`
5. 可选：绑定自定义域名 + 开通 CDN 加速

---

## 四、方式 B：接入云端数据（多端同步）

如需「多设备同步」「数据备份」「换浏览器不丢数据」，需要加后端和数据库。

### 架构示意

```
用户浏览器  ←→  静态站点(OSS/CDN)  +  后端 API(云服务器/Serverless)  ←→  数据库
```

### 国内云组合示例

| 组合 | 前端 | 后端 | 数据库 | 备注 |
|------|------|------|--------|------|
| **1** | 阿里云 OSS + CDN | 阿里云 ECS | 云数据库 RDS | 常规方案 |
| **2** | 腾讯云 COS + CDN | 腾讯云 SCF(Serverless) | 云开发 CloudBase | 免运维 |
| **3** | 七牛云 | 阿里云 函数计算 FC | 云数据库 | 低成本 |
| **4** | 任意静态托管 | 腾讯云 云开发 | 云开发自带数据库 | 上手快 |

### 需要做的开发

1. **后端 API**（Node/Python/Go 等）：  
   - 提供清单、备忘录的 CRUD 接口  
   - 使用 JWT 或 Session 做用户鉴权

2. **数据库**：  
   - MySQL / PostgreSQL（RDS）  
   - 或 MongoDB（文档型）

3. **修改前端**：  
   - 在 `storage.js` 中改为调用上述 API，不再使用 `localStorage`

4. **配置**：  
   - 在 `config.js` 中设置 `API_BASE` 为你的 API 地址

---

## 五、推荐路径

- **个人使用、演示**：选 **方式 A**，用七牛云/阿里 OSS 托管静态文件，成本低。
- **需要多端同步**：选 **方式 B**，优先考虑腾讯云云开发或阿里云「函数计算 + RDS」，开发量适中。

---

## 六、快速部署检查清单（方式 A）

- [ ] 将 `daily-checklist-app` 下的所有文件上传到 OSS/COS 等
- [ ] 确认目录结构：`index.html` 与 `css/`、`js/` 同目录
- [ ] 开启静态网站托管，默认首页为 `index.html`
- [ ] 如需域名：在控制台绑定域名并配置 CDN
- [ ] 使用 `https://` 访问（部分浏览器对 `file://` 有限制）
